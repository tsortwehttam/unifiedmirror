## Config Behavior

Default config root is `.um`.

Read resolution order:

1. `UM_CONFIG_DIR`, if set
2. local `.um`
3. local `.msgmon`
4. install-local `.um`
5. install-local `.msgmon`
6. home `~/.um`
7. home `~/.msgmon`

Write target:

- always `.um` by default unless `UM_CONFIG_DIR` is set

This means existing `msgmon` credentials are readable, but new writes go to `.um`.

### Gmail Pull After Query Fix

Command used:

```bash
um pull \
  --platform gmail \
  --account mrtrost@gmail.com \
  --query 'in:inbox' \
  --since 2026-03-17T00:00:00Z \
  --until 2026-03-22T23:59:59Z \
  --dest /tmp/um-inbox-test.jsonl \
  --verbose
```

Observed behavior:

- upstream Gmail query became:
  - `in:inbox after:1773705600 before:1774224000`
- result:
  - wrote `39` messages

Conclusion:

- Gmail pull is working for this account with `in:inbox`
- the time-window narrowing fix is working
